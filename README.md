# Help Mobility — website (faithful rebuild)

A fast, accessible rebuild of [Help Mobility](https://helpmobility.ca) — mobility &
home‑healthcare equipment (sales, rentals & repairs) for the Greater Toronto Area.
It keeps the **real brand**: the actual logo, the brand colours (blue `#004678`
“Help” + green `#00780a` “Mobility”) and real photography from the live site — while
**fixing the issues** found on the original. See **[ISSUES.md](ISSUES.md)** for the
full flagged‑and‑fixed list.

It’s a single‑codebase **Progressive Web App (PWA)**: it runs in any browser on
desktop and phones, and can be *installed* like a native app on Windows, macOS,
Android and iOS — no app store required. No build step, no dependencies.

> The real Help Mobility is **consultation / quote‑driven** and does not publish
> prices. This rebuild reflects that honestly: it captures interest into a **quote
> request** (a real, contactable lead) instead of showing invented prices, ratings
> or reviews.

---

## What’s on the site

- **Home** — hero, mission, the four product categories, rentals & funding highlights.
- **Explore Products** — the four categories (Mobility, Accessibility, Bathroom, Daily
  Living) mapped to the store’s **real collections**, with search and category filters.
- **Rentals** — the real rental line‑up (hospital beds, stairlifts, scooters, wheelchairs…).
- **Repairs & Maintenance** — real services (same‑day repairs, loaner equipment, 7 days a week…).
- **Industries We Serve** — hospitals, therapists, long‑term care, retirement & senior communities.
- **Funding & Assistance** — ADP, ODSP, Ontario Works, WSIB, March of Dimes, Spinal Cord Injury Ontario.
- **Contact** + **Quote request** — capture a lead by phone, form, or an “add to quote” list.

The real, verified phone number **905‑615‑9302** is used site‑wide as a clickable link.

## Run it

It’s static — **no build step, no dependencies.**

**Quickest:** open `index.html` directly in a browser. *(Browsing works from
`file://`; PWA install/offline needs `http(s)` — use a local server below.)*

```bash
# any one of these from the project folder:
python3 -m http.server 8080
# or
npx serve .
```

Then visit **http://localhost:8080**.

### Install as an app
- **Desktop (Chrome/Edge):** open the site → click the **Install** icon in the address bar.
- **Android (Chrome):** menu → **Add to Home screen / Install app**.
- **iOS (Safari):** Share → **Add to Home Screen**.

## Customize

### Content & company details
Everything customer‑facing lives in **`assets/js/data.js`**:

- `COMPANY` — name, tagline, mission, region, phone.
- `CATEGORIES` — the four sections, each with its image and list of real collections.
- `RENTALS`, `REPAIRS`, `INDUSTRIES`, `FUNDING` — the corresponding pages.

### Real product catalogue
This rebuild presents the real category → collection **structure** and turns interest
into quote requests. To list individual products with photos and details, add a
`products` array per collection in `data.js` and render product cards in `app.js`.

---

## Go live: leads, analytics, reviews, landing pages

Open **`assets/js/data.js`** and fill in `CONFIG` / `REVIEWS` — everything is wired:

- **`CONFIG.leadEndpoint`** — paste a Formspree / Getform / Google Apps Script / CRM
  webhook URL and every quote, contact and callback is POSTed to it as JSON (leads
  reach you instantly). Until set, leads are stored in the browser; run
  `HM.exportLeads()` in the DevTools console to review them.
- **`CONFIG.gaId`** — paste a GA4 Measurement ID (`G-…`) to load analytics and track
  conversions: a `generate_lead` event on each quote/callback and a `contact` event
  on each phone‑number tap.
- **`REVIEWS`** — add genuine customer reviews and a testimonials section appears
  automatically (left empty by design — no fabricated reviews).

### SEO landing pages

Standalone, crawlable pages for each product category (`/products/<cat>/`) and GTA
city (`/areas/<city>/`) are generated by:

```bash
node tools/build-landing.js   # also refreshes sitemap.xml
```

Edit the `CITIES` list in `tools/build-landing.js` to add more service areas.

See **[GROWTH.md](GROWTH.md)** for the full conversion/SEO playbook and next steps.

## Deploy (free options)

- **GitHub Pages:** push to GitHub → Settings → Pages → deploy from branch. Paths are
  relative, so it works from a project subpath (`/desktop-tutorial/`).
- **Netlify / Vercel / Cloudflare Pages:** drag‑and‑drop the folder or connect the repo.
  No build command; publish directory = project root.

## Accessibility

Designed for an older / caregiver audience: 17–18px base text, AA‑contrast colours,
≥44px tap targets, visible focus rings, keyboard‑navigable, `aria-current` on the
active nav, `prefers-reduced-motion` respected, and semantic landmarks with a skip link.
